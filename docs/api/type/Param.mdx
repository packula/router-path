# `Param`

The interface implemented by all route params.

## Properties {#properties}

:::info

All properties are read-only.

:::

### `name` {#property-name}

The name of the param, as a string of type [`Name`][name type].

The `name` property determines which arg gets used for this param when building
a path string, as well as which property of the result will contain the value
parsed for this param when matching against a path string.

### `exp` {#property-exp}

A [regular expression] with a [capturing group], used to match and extract a
portion of a path string for subsequent parsing as the value of this param.

When successfully matched, the content from the [capturing group] will be passed
to [`parse`] for parsing.

:::caution

Param expressions **must** use exactly 1 [capturing group], otherwise errors
will be thrown when attempting to match against path strings. This limitation
helps limit complexity in the matching system.

:::

## Methods {#methods}

### `build` {#method-build}

Builds the string representation of this param used when building path strings.

#### Parameters {#method-build-parameters}

`build` should have a single parameter of type [`Arg`], which accepts the
"value" of the param.

#### Return value {#method-build-return-value}

`build` should return a string representation of this param.

:::tip

The strings produced by `build` should match [`exp`], and the resulting
[capturing group] content should be able to be parsed by [`parse`].

:::

### `parse` {#method-parse}

Parses a successful match against [`exp`].

#### Parameters {#method-parse-parameters}

`parse` should have a single string parameter, which accepts the
[capturing group] content of a successful match against [`exp`].

:::tip

`parse` should be capable of parsing *any* [capturing group] content that can
occur in a successful match of [`exp`] against the strings that are produced by
[`build`].

:::

#### Return value {#method-parse-return-value}

`parse` should return a value of type [`Result`] which represents the parsed
"value" of the param.

## Type variables {#type-variables}

The `Param` type is a [generic type]:

    Param<Name extends string, Arg = string, Result = Arg>

### `Name` {#type-variable-name}

The param [`name`] as a [literal type].

### `Arg` {#type-variable-arg}

The type of the arg that must be supplied for this param when building a path
string.

Leaving off both the `Arg` and `Result` type variables creates a `Param` where
both the `Arg` and `Result` types are `string`, meaning that the following types
are equivalent:

```ts no-transpile
import {Param} from '@snout/router-path'

type ParamA = Param<'param-name', string, string>
type ParamB = Param<'param-name', string>
type ParamC = Param<'param-name'>
```

### `Result` {#type-variable-result}

The type of the value that will be parsed for this param when matching against a
path string.

Leaving off the `Result` type variable creates a `Param` type where the `Arg`
and `Result` types are the same, meaning that the following types are
equivalent:

```ts no-transpile
import {Param} from '@snout/router-path'

type ParamA = Param<'param-name', boolean, boolean>
type ParamB = Param<'param-name', boolean>
```

## Usage {#usage}

When writing custom param creators, the `Param` type helps define the types
that are accepted when building, and parsed when matching:

```ts no-transpile
import {Param, path} from '@snout/router-path'

/**
 * Creates a param that takes any type of arg, but only produces a string
 */
function looseParam<Name extends string> (name: Name): Param<Name, any, string> {
  return {
    name,
    exp: /([^/]+)/,
    build: arg => String(arg),
    parse: match => match,
  }
}

const pathA = path`/a/${looseParam('b')}/c`
pathA.build({b: 'x'}) // returns '/a/x/c'
pathA.match('/a/y/b') // returns {b: 'y'}
```
